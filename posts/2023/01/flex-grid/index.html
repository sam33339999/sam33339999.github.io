Flex vs Grid

Flex box 常用於小規模佈局； Grid layout則適合大規模佈局。

  • Flex box 是單向度的排版。
  • Grid 則是包含 X, Y 軸的平面排版方式。

如果想要簡單輕便的實現單一軸向的自適應排版(RWD)，例如 導覽列, 卡片式排版, 相簿 等… 可以選擇使用 Flexbox

如果你想要快速實現雙軸平面式，甚至不規則的排版（像 平面設計排版 那樣的自由) 那麼使用 Grid layout 會更適合也更快速。

Flexbox 簡介

Flexbox layout 是基於 display: flex 或是 display: inline-flex 的語法， 以單向主軸或交叉軸為方向的排版。

這個語法透過 flex-flow 的各項數值能輕易的實現適應式排版， flex-flow 能決定排版以實際物理的方向 (上/右/下/左)、 直行或橫列排版 ( row / column ) 以及 Flex item 遇到 Flex Container 的寬度極限時，超出或是換行排列。 flex 規定元素如何伸長或縮短以適應 flex 容器中的可用空間。

flex 外元素

主軸 Main Axis 與 交叉軸 Cross Axis -> 如何定義 主軸與交叉軸 -> 會跟據 flex-direction 的流向，才去了解當下的主軸、交叉軸是 左至右｜ 右至左｜ 上至下｜ 下至上

  • display: flex, inline-flex
  • flex-flow: flex 的排版流向 (flex items) 下方兩個屬性的簡寫 … 可以寫成 flex-flow: row wrap;
    • flex-direction: flex 的流向 … row, row-reverse, column, column-reverse
    • flex-wrap: 超出範圍時是否換行屬性，單行顯示還是多行顯示 … nowrap, wrap, wrap-reverse … 默認: nowrap
  • justify-content:
    • flex-start主軸的起點
    • flex-end主軸的終點
    • center主軸置中
    • space-between 左右貼齊平均分配
    • space-around 不貼齊左右平均分配
  • align-items: 交叉軸的對齊設定
    • flex-start 交叉軸的起點
    • flex-end 交叉軸的終點
    • center 交叉軸置中
    • stretch 拉伸(拉長內部容器至交錯軸左右)
    • baseline 基準線，把內部元素的文字拉成一個基準線水平
  • align-content: 排列內容（多行版本，如果 flex 的內元素過多，會靠哪邊？），注意 stretch 在高度被限制的情況下不會正常拉伸
    • flex-start: 靠主軸頭
    • flex-end: 靠主軸尾
    • center: 集中在中間
    • space-between: 分布在兩側
    • space-around: 平均排列
    • stretch: 拉伸

flex 內元素

  • flex: 可將下面三個屬性寫成一塊，例如: flex: 0 1 auto
    • flex-grow: 是數值，flex 增長係數，默認為 0 不能為負數，當內部元素小於外層容器大小，去做增長。
    • flex-shrink: 是數值，指定 flex 收縮規則，flex 元素僅在默認寬度之和大於容器的時候才會發生收縮，其收縮大小依照此設定而變化，
    • flex-basis: 是長度單位數值|auto，表示其預設分配到的空間，跟 width 很像；但優先度比較高。 默認為 auto
    • Day24 Flex 空間分配 flex-grow / flex-shrink / flex-basis
  • order: 排序，是數值；可以負數。
  • align-self: stretch, center, start, end, self-start, self-end, flex-start, flex-end
