swoole-on-type

swoole 簡介

基於 Zend VM 去撰寫的擴充、拓展 (extension)，使得 PHP 能夠擁抱現今眾多的新特性… 如: 異步 I/O， 協程， 多執行緒。

  • swoole v4.* 實現了 coroutine (async)
  • swoole v5.* goroutine

swoole server mode

  • Base Mode -> 囤積請求的感覺，因為沒有上層的進程保護，故若是程序死掉，囤積得請求會一併的遺失。（由於無保護，故速度會快一些）
  • Process Mode -> 有上層保護，故丟失請求的機率會降低

說明

  • PHP - 實現 SAPI (System API) -> 來去使 Zend VM 與操作系統達成對應的事務。
  • PHP script 將透過 PHP Zend VM 去執行。
    • 由於多了 Zend VM 去執行，故才可以使用同一套 script 達成多系統間使用同腳本執行

為什麼 PHP 慢？

  • Blocking I/O -> 阻塞 I/O
    • 實踐相對簡單
  • Stateless -> 無狀態
    • 由於無狀態，因此多數 PHPer 會在開發的當下可能忽略了 memory leak 的問題

網路模型，Http 也是基於 TCP 模型，由上述例子中，我們針對 request 去做他的 handler ，並且回傳一個字串 Hello Swoole!。 那麼；為什麼沒有寫出其他的狀態的 function，原因為我們 web 開發者比較會專注於 request 去做對應的處理
